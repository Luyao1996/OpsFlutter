## 1. 系统架构概览

### 1.1 当前系统状态

- **netbar-ops-api**: 后端 API 服务，目前使用假数据
- **netbar-ops-client**: 待开发的数据采集程序，部署在各终端/网吧

### 1.2 数据流向

```
┌─────────────────┐     HTTP/WebSocket      ┌─────────────────┐
│                 │ ─────────────────────▶  │                 │
│  采集客户端      │                         │  netbar-ops-api │
│  (终端机器)      │ ◀─────────────────────  │    (服务端)      │
│                 │     命令下发             │                 │
└─────────────────┘                         └─────────────────┘
```

### 1.3 部署模式

- **终端级部署**: 每台终端机器运行一个采集客户端
- **网吧级部署**: 每个网吧有一个主控节点汇总数据（可选）

## 2. 需要采集的数据

### 2.1 终端基础信息（一次性/变更时上报）

| 字段 | 类型 | 说明 | 采集方式 |
|------|------|------|---------|
| name | string | 终端名称 | 配置文件/自动生成 |
| code | string | 终端编码（唯一） | 配置文件/MAC生成 |
| ip | string | IP地址 | 系统API获取 |
| mac | string | MAC地址 | 系统API获取 |
| os | string | 操作系统 | 系统API获取 |
| type | string | 类型(pc/server/console/cashier) | 配置文件 |
| netbar_id | uint | 所属网吧ID | 配置文件 |
| area_id | uint | 所属区域ID | 配置文件（可选） |

### 2.2 终端状态信息（定时上报，建议间隔 10-30 秒）

| 字段 | 类型 | 说明 | 采集方式 |
|------|------|------|---------|
| status | int | 状态: 0=离线, 1=空闲, 2=使用中 | 用户会话检测 |
| cpu_usage | float64 | CPU使用率(0-100) | 系统API |
| ram_usage | float64 | 内存使用率(0-100) | 系统API |
| gpu_usage | float64 | GPU使用率(0-100) | NVIDIA/AMD API |
| disk_usage | float64 | 磁盘使用率(0-100) | 系统API |
| uptime | string | 运行时长(如"5天3小时") | 系统API计算 |

### 2.3 启动项执行统计（事件触发上报）

| 字段 | 类型 | 说明 | 采集时机 |
|------|------|------|---------|
| startup_item_id | uint | 启动项ID | 启动时 |
| launch_count | int | 启动次数 | 每次启动+1 |
| failure_count | int | 失败次数 | 启动失败+1 |
| survival_1min | int | 存活<1分钟次数 | 进程退出时判断 |
| survival_10min | int | 存活<10分钟次数 | 进程退出时判断 |
| survival_20min | int | 存活<20分钟次数 | 进程退出时判断 |

### 2.4 网吧汇总信息（网吧级采集，定时上报）

| 字段 | 类型 | 说明 | 采集方式 |
|------|------|------|---------|
| netbar_id | uint | 网吧ID | 配置 |
| online_seats | int | 在线座位数 | 汇总终端状态 |
| status | int | 1=在线, 0=离线 | 主控节点状态 |

### 2.5 系统日志（事件触发上报）

| 字段 | 类型 | 说明 |
|------|------|------|
| level | string | 日志级别: info/warn/error |
| module | string | 模块名: terminal/startup/system |
| action | string | 操作名 |
| message | string | 日志消息 |

## 3. 现有可用接口

### 3.1 终端管理接口

#### 获取终端列表
```
GET /api/v1/terminals
Query: search, netbar_id, status, type
Response: Terminal[]
```

#### 获取单个终端
```
GET /api/v1/terminals/:id
Response: Terminal
```

#### 创建终端（客户端首次注册）
```
POST /api/v1/terminals
Body: {
    "name": "PC-001",
    "code": "T-NB001-001",
    "netbar_id": 1,
    "ip": "192.168.1.100",
    "mac": "AA:BB:CC:DD:EE:FF",
    "os": "Windows 10",
    "type": "pc"
}
Response: Terminal
```

#### 更新终端信息
```
PUT /api/v1/terminals/:id
Body: {
    "status": 1,
    "cpu_usage": 45.6,
    "ram_usage": 62.3,
    ...
}
Response: Terminal
```

#### 获取终端心跳状态
```
GET /api/v1/terminals/:id/heartbeat
Response: {
    "terminal_id": 1,
    "status": 1,
    "cpu_usage": 45.6,
    "ram_usage": 62.3,
    "gpu_usage": 28.5,
    "disk_usage": 71.2,
    "uptime": "5天3小时",
    "last_heartbeat": "2024-01-27T10:30:00Z"
}
```

#### 远程操作（服务端下发）
```
POST /api/v1/terminals/:id/remote
Body: { "action": "restart" | "shutdown" | "screenshot" | "wake" }
Response: { "message": "...", "terminal_id": 1, "action": "restart" }
```

### 3.2 启动项接口

#### 获取启动项列表
```
GET /api/v1/startup-items
Query: zone, netbar_id, enabled, search
Response: StartupItem[]
```

#### 获取启动项监控统计
```
GET /api/v1/monitor/startup-items  或  GET /api/v1/startup-items/monitor
Response: StartupItemStats[]
```

### 3.3 资源文件接口

#### 下载资源文件
```
GET /api/v1/resources/:id/download
Response: 文件流
```

#### 下载目录（打包）
```
GET /api/v1/resources/:id/download-dir
Response: 压缩包
```

### 3.4 认证接口

#### 登录（获取 Token）
```
POST /api/v1/auth/login
Body: { "username": "...", "password": "..." }
Response: { "token": "...", "user": {...} }
```

### 3.5 日志接口

#### 创建日志（当前无此接口，见缺少接口部分）
目前只有查询日志接口：
```
GET /api/v1/logs
Query: level, module, search, start_date, end_date, page, page_size
```

## 4. 缺少的接口（需要新增）

### 4.1 终端心跳上报接口 ⭐ 关键

**当前状态**: 只有 `GET /api/v1/terminals/:id/heartbeat` 用于查询，缺少上报接口

**建议新增**:
```
POST /api/v1/terminals/:id/heartbeat
Body: {
    "status": 2,
    "cpu_usage": 45.6,
    "ram_usage": 62.3,
    "gpu_usage": 28.5,
    "disk_usage": 71.2,
    "uptime": "5天3小时"
}
Response: { "message": "ok", "next_interval": 30 }
```

### 4.2 终端注册接口 ⭐ 关键

**当前状态**: 需要通过 `POST /api/v1/terminals` 创建，但这是管理接口

**建议新增**（专门用于客户端自动注册）:
```
POST /api/v1/client/register
Body: {
    "netbar_code": "NB001",       // 通过网吧编码关联
    "mac": "AA:BB:CC:DD:EE:FF",
    "ip": "192.168.1.100",
    "hostname": "PC-001",
    "os": "Windows 10 Pro",
    "os_version": "10.0.19045",
    "type": "pc"
}
Response: {
    "terminal_id": 123,
    "terminal_code": "T-NB001-001",
    "token": "client_token_xxx",    // 客户端专用token
    "config": {
        "heartbeat_interval": 30,

        "resources_to_sync": [...]

}
```

## 5. 数据采集方案建议

### 5.1 客户端架构

```
┌─────────────────────────────────────────────────────────┐
│                    采集客户端                            │
├─────────────────────────────────────────────────────────┤
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐     │
│  │ 系统信息采集 │  │ 进程监控模块 │  │ 启动项管理  │     │
│  └─────────────┘  └─────────────┘  └─────────────┘     │
│         │                │                │             │
│         ▼                ▼                ▼             │
│  ┌─────────────────────────────────────────────────┐   │
│  │              数据聚合 & 缓存层                    │   │
│  └─────────────────────────────────────────────────┘   │
│                          │                              │
│         ┌────────────────┼────────────────┐            │
│         ▼                ▼                ▼            │
│  ┌───────────┐    ┌───────────┐    ┌───────────┐      │
│  │ HTTP 客户端│    │ WebSocket │    │ 本地存储  │      │
│  └───────────┘    └───────────┘    └───────────┘      │
└─────────────────────────────────────────────────────────┘
```

### 5.2 采集流程

```
1. 客户端启动
   │
   ├─▶ 读取本地配置
   │
   ├─▶ 检查是否已注册
   │   ├─ 否 ──▶ 调用注册接口 ──▶ 保存终端ID和Token
   │   └─ 是 ──▶ 继续
   │
   ├─▶ 建立 WebSocket 连接（或使用 HTTP 轮询）
   │
   ├─▶ 获取最新配置（启动项、资源列表）
   │
   ├─▶ 同步所需资源文件
   │
   └─▶ 进入主循环
       │
       ├─▶ 定时采集系统信息（每10-30秒）
       │   └─▶ 上报心跳数据
       │
       ├─▶ 监控启动项进程
       │   └─▶ 上报启动项状态变更
       │
       └─▶ 监听服务端命令
           └─▶ 执行远程操作并回报结果
```

### 5.3 离线处理

```
网络断开时:
1. 数据缓存到本地 SQLite/文件
2. 定期尝试重连
3. 恢复连接后批量上报缓存数据
```

### 5.4 安全考虑

1. **认证**: 使用专用的客户端 Token，与用户 Token 分离
2. **通信**: 建议使用 HTTPS/WSS
3. **数据校验**: 服务端验证终端ID与Token的对应关系
4. **防重放**: WebSocket 消息带时间戳，HTTP 请求带 nonce

## 6. 数据模型参考

```go
type Terminal struct {
    ID            uint       `json:"id"`

    RAMUsage      float64    `json:"ram_usage"`      // 0-100
    GPUUsage      float64    `json:"gpu_usage"`      // 0-100

    Uptime        string     `json:"uptime"`         // "5天3小时"
    LastOnline    *time.Time `json:"last_online"`
    LastHeartbeat *time.Time `json:"last_heartbeat"`
}
```
