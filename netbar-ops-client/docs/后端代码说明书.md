## 二、核心架构

### 2.1 统一响应格式

所有API通过 `ResponseTrait` 返回统一格式：

```php
// 位置: app/Controller/ResponseTrait.php

// 成功响应
return $this->response(['key' => 'value']);
// 输出: {"code": 0, "message": "success", "data": {"key": "value"}}

// 成功消息
return $this->success('操作成功');
// 输出: {"code": 0, "message": "操作成功", "data": null}

// 失败响应
return $this->failed('错误信息');
// 输出: {"code": 1, "message": "错误信息", "data": null}
```

### 2.2 路由定义方式

项目使用**注解路由**，在Controller类和方法上定义：

```php
#[Controller(prefix: 'api/user')]  // 控制器前缀
#[Middleware(AuthUserMiddleware::class)]  // 中间件

class UserController extends AbstractController
{
    #[RequestMapping(path: '', methods: RequestMapping::GET)]  // GET /api/user
    public function index() { ... }

    #[RequestMapping(path: '{id:\d+}', methods: RequestMapping::GET)]  // GET /api/user/{id}
    public function show(int $id) { ... }

    #[RequestMapping(path: '{id:\d+}', methods: [RequestMapping::PUT, RequestMapping::POST])]
    public function update(int $id) { ... }
}
```

### 2.3 认证机制

**JWT Token认证**:
- 令牌通过 `Authorization: Bearer <token>` 请求头传递
- 两种Guard: `api` (用户) 和 `merchant` (商户)
- 令牌有过期和刷新机制

**中间件**:
- `AuthUserMiddleware` - 用户认证
- `AuthMerchantMiddleware` - 商户认证
- `ValidationMiddleware` - 参数验证
- `CorsMiddleware` - 跨域请求

**位置**: `app/Controller/Api/PassportController.php`
**前缀**: `/api/passport`

| 方法 | 路由 | 功能 | 认证 |
|------|------|------|------|
| login | POST / | 预登录，返回pwd和二维码 | 否 |

1. 前端调用 `/login` 获取 `pwd` 和二维码

### 5.2 典型服务实现

```php
// UserService 示例
class UserService
{
    public function saveOrUpdate(User $operator, array $data, ?int $id = null): bool
    {
        // 权限检查
        if (!$operator->isTopManager && !$operator->isSubManager) {
            throw new \Exception('无权完成操作');
        }

        // 创建或更新用户
        if ($id) {
            $user = User::find($id);
            // 更新逻辑...
        } else {
            $user = new User();
            // 创建逻辑...
        }

        // 角色同步
        if (isset($data['role_ids'])) {
            $user->roles()->sync($data['role_ids']);
        }

        return $user->save();
    }
}
```

## 七、权限控制逻辑

### 7.1 用户角色层级

```
总部管理员 (group_id = 0)
├── 可管理所有分组
├── 可管理所有商户
├── 可管理所有用户
└── 可配置系统设置

分组管理员 (group_id > 0 && is_manager = true)
├── 仅管理本分组
├── 仅管理本分组下的商户
├── 仅管理本分组下的用户
└── 不可访问其他分组

普通用户 (group_id > 0 && is_manager = false)
├── 仅访问授权的商户
└── 有限的操作权限
```

### 7.2 权限检查示例

```php
// 控制器中常见的权限检查
$user = $this->auth->guard('api')->user();

if (!$user->isTopManager) {
    // 非总部管理员，限制查询范围
    $query->where('group_id', $user->group_id);
}

if ($user->isNormalUser) {
    // 普通用户，进一步限制
    $query->whereHas('users', function ($q) use ($user) {
        $q->where('id', $user->id);
    });
}
```
