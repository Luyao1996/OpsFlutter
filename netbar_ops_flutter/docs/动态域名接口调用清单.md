# 动态域名接口调用清单

> 本文档梳理 Flutter 端所有依赖网吧动态域名（`subdomainFull`）的接口调用，覆盖 HTTP REST、WebSocket、浏览器跳转三类形态。
> 示例域名统一使用 `7gzpa8j3z9by.frps.wwls.net`；`{seat}` 表示座位编号（如 `001`）。

## 九、URL 形态汇总

| 形态 | 模板 | 场景数 |
|---|---|---|
| ① REST `https://{full}/api/...` | TerminalApi 12 个方法 + ScreenshotApi + 桌面页 `/api/seatlist` | 14 |
| ② REST `https://router-{hostOnly}/api/...` | RouterApi 6 个方法 | 6 |
| ③ WS `ws://{full}/ws_client` | 控制台 CMD（4 类帧） | 1 连接 |
| ④ noVnc 跳转 host 参数含 `{full.split('.').first}` | VNC 浏览器跳转 | 1 |
| ⑤ WebRTC 信令 Peer 参数含 `{full.split('.').first}` | WebRTC 远程桌面 | 1 |

### URL 衍生关系图

```
CurrentNetbar.subdomainFull ──┬──► ① https://{full}/api/...           [TerminalApi / DesktopApi]
        (例: x.frps.wwls.net) │
                              ├──► hostOnly = full.split(':')[0]
                              │       └──► ② https://router-{hostOnly}/api/...    [RouterApi]
                              │
                              ├──► full去http前缀
                              │       └──► ③ ws://{full}/ws_client              [Console CMD]
                              │
                              └──► subFirst = full.split('.')[0]
                                      ├──► ④ noVnc URL host={seat}-{subFirst}   [VNC]
                                      └──► ⑤ wss WebRTC Peer={seat}-{subFirst}  [WebRTC]
```

## 十、关键约束 / 陷阱

1. **Host header 必须保留原大小写**：`terminal_api.dart:67,88` 显式注入 `Host: domain.trim()`；raw socket helper 也做同样处理（`terminal_api_http_helper_io.dart:53,172`）。说明后端对 Host 区分大小写或对 frp 网关有依赖。
2. **自签名证书放通**：`terminal_api.dart:42-48` 网吧本地后端使用自签证书。
3. **无代理**：`findProxy = (uri) => 'DIRECT'`，避免企业代理污染网吧域名解析。
4. **超时 60s**：网吧本地链路 RTT 较长，统一设 60s（路由器代理 30s）。
5. **404/CORS 静默降级**：`_shouldReturnEmpty()` 把 404/405/501/CORS/network error 转空列表，不让网吧离线把整个 UI 打挂。
6. **空 domain 早返回**：所有 API 都做 `if (domain == null || domain.isEmpty) return ...` 守卫。
7. **netbarId family + `_ownerNetbarId` 双锁**：防止切换网吧后旧域名串台到新页面（commit `911343f`、`880213d`）。
